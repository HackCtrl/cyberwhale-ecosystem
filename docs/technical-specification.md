
# CyberWhale Platform - Техническая спецификация

## 1. Введение

### 1.1 Назначение документа
Данный документ представляет собой техническую спецификацию платформы CyberWhale, образовательной экосистемы в области кибербезопасности. Документ содержит исчерпывающую информацию о технической реализации, архитектуре, используемых технологиях и методологиях разработки.

### 1.2 Область применения
Техническая спецификация предназначена для технических специалистов, разработчиков, архитекторов систем и других заинтересованных сторон, участвующих в разработке, поддержке и развитии платформы CyberWhale.

### 1.3 Термины и определения
- **CTF** (Capture The Flag) - соревнования по информационной безопасности
- **RLS** (Row Level Security) - механизм безопасности на уровне строк в базе данных
- **JWT** (JSON Web Token) - стандарт для создания токенов доступа
- **SPA** (Single Page Application) - одностраничное веб-приложение
- **API** (Application Programming Interface) - интерфейс программирования приложений

## 2. Общее описание системы

### 2.1 Назначение системы
Платформа CyberWhale представляет собой образовательную экосистему, направленную на обучение и практику в области кибербезопасности. Система предоставляет интерактивные обучающие материалы, практические задания в формате CTF, сообщество для обсуждения и инструменты для развития навыков в сфере информационной безопасности.

### 2.2 Целевая аудитория
- Начинающие специалисты в области кибербезопасности
- Опытные профессионалы, желающие углубить свои знания
- Студенты профильных специальностей
- Преподаватели и инструкторы по кибербезопасности
- ИТ-специалисты, желающие расширить свои компетенции

### 2.3 Функциональные возможности
- Обучающие материалы и курсы по кибербезопасности
- Интерактивные практические задания (CTF-платформа)
- База знаний с справочными материалами
- ИИ-ассистент для персонализированного обучения
- Профессиональное сообщество
- Система отслеживания прогресса и достижений

## 3. Архитектура системы

### 3.1 Общая архитектура
Система реализована на основе клиент-серверной архитектуры с четким разделением ответственности между компонентами:

![Диаграмма архитектуры](./architecture/architecture_diagram.png)

### 3.2 Компоненты системы

#### 3.2.1 Фронтенд
- React-приложение с маршрутизацией
- Компоненты пользовательского интерфейса
- Управление состоянием приложения
- Интеграция с бэкенд-сервисами

#### 3.2.2 Бэкенд
- База данных PostgreSQL через Supabase
- Аутентификация и авторизация на основе JWT
- Serverless-функции для обработки логики
- API для взаимодействия с клиентом

#### 3.2.3 Инфраструктура
- Хостинг на платформе Lovable
- Непрерывная интеграция и доставка (CI/CD)
- Мониторинг и логирование
- Управление секретами и конфигурациями

### 3.3 Схема данных
Система использует реляционную базу данных PostgreSQL со следующими основными сущностями:
- Пользователи (profiles)
- Учебные материалы (articles)
- CTF-задания (challenges)
- Достижения (achievements)
- Прогресс пользователя (user_progress)

### 3.4 Интеграции
- Supabase для базы данных, аутентификации и хранилища
- ИИ-сервисы для работы ассистента
- Интеграция с внешними обучающими ресурсами

## 4. Технологический стек

### 4.1 Фронтенд

#### 4.1.1 Основной фреймворк
- **React** (v18.3.1): Компонентный подход к построению пользовательского интерфейса
- **TypeScript** (v5.0.0+): Строгая типизация для улучшения качества кода
- **Vite**: Сборка проекта с горячей перезагрузкой модулей и оптимизацией для производства

#### 4.1.2 Компоненты и стилизация
- **Tailwind CSS**: Утилитарный CSS-фреймворк для быстрой разработки UI
- **Shadcn UI**: Библиотека компонентов, построенная на Radix UI
- **Lucide Icons**: Библиотека иконок с единым дизайном
- **Framer Motion**: Библиотека для создания анимаций

#### 4.1.3 Управление состоянием и получение данных
- **React Context API**: Глобальное управление состоянием
- **Tanstack Query** (v5.56.2): Синхронизация, получение и обновление серверных данных
- **Zod** (v3.23.8): Валидация схем данных

#### 4.1.4 Маршрутизация
- **React Router Dom** (v6.26.2): Декларативная маршрутизация для React-приложений

### 4.2 Бэкенд

#### 4.2.1 База данных и аутентификация
- **Supabase**: Поставщик PostgreSQL базы данных, аутентификации, хранилища и serverless-функций
- **PostgreSQL**: Реляционная база данных
- **Row Level Security (RLS)**: Политики безопасности на уровне базы данных

#### 4.2.2 Serverless-функции
- **Supabase Edge Functions**: Serverless-функции на основе Deno
- **TypeScript/Deno Runtime**: Среда выполнения для edge-функций

#### 4.2.3 Хранилище
- **Supabase Storage**: Сервис хранения объектов
- **CDN**: Для быстрой доставки статического контента

### 4.3 Инструменты разработки

#### 4.3.1 Качество и согласованность кода
- **ESLint**: Инструмент для обнаружения и исправления проблем в коде
- **Prettier**: Форматирование кода для обеспечения единого стиля
- **TypeScript Compiler**: Статический анализ кода

#### 4.3.2 Тестирование
- **Vitest**: Фреймворк для юнит-тестирования
- **Testing Library**: Утилиты для тестирования компонентов
- **Playwright**: Фреймворк для end-to-end тестирования

#### 4.3.3 Контроль версий и совместная работа
- **Git**: Распределенная система контроля версий
- **GitHub**: Платформа для хостинга репозиториев и совместной работы

### 4.4 DevOps и инфраструктура

#### 4.4.1 Развертывание и хостинг
- **Lovable Platform**: Платформа хостинга и развертывания
- **Continuous Integration**: Автоматизированное тестирование и проверка сборки
- **Continuous Deployment**: Автоматизированный процесс развертывания

#### 4.4.2 Мониторинг и аналитика
- **Отслеживание ошибок**: Захват и анализ ошибок времени выполнения
- **Мониторинг производительности**: Отслеживание метрик производительности
- **Аналитика использования**: Понимание поведения пользователей

## 5. Безопасность

### 5.1 Защита данных
- **HTTPS/TLS**: Безопасная передача данных между клиентом и сервером
- **JWT**: Безопасный метод представления утверждений между сторонами
- **Хеширование паролей**: Безопасное хранение учетных данных с использованием bcrypt
- **Политики CORS**: Контролируемый доступ к ресурсам между доменами

### 5.2 Безопасность приложения
- **Content Security Policy**: Защита от XSS-атак
- **Rate Limiting**: Предотвращение брутфорс-атак и DoS
- **Валидация ввода**: Защита от атак внедрения и некорректных данных
- **Регулярные аудиты безопасности**: Систематический обзор мер безопасности

### 5.3 Аутентификация и авторизация
- **Многофакторная аутентификация**: Дополнительный уровень защиты
- **Управление сессиями**: Безопасное управление пользовательскими сессиями
- **Ролевой доступ**: Контроль доступа на основе ролей пользователей

## 6. Требования к окружению

### 6.1 Требования для разработки
- **Node.js**: версия 18.0.0 или выше
- **npm**: версия 9.0.0 или выше (или Yarn 1.22.0+)
- **Git**: версия 2.30.0 или выше
- **VS Code** (рекомендуемый редактор)
- **Supabase CLI** (для локальной разработки с бэкендом)

### 6.2 Требования для развертывания
- **Lovable аккаунт**: Для хостинга и развертывания
- **Supabase аккаунт**: Для бэкенд-сервисов
- **Доменное имя** (опционально): Для настройки пользовательского домена

### 6.3 Требования к браузеру
- **Chrome**: версия 90 или выше
- **Firefox**: версия 90 или выше
- **Safari**: версия 14 или выше
- **Edge**: версия 90 или выше

## 7. Процедуры разработки

### 7.1 Методология разработки
Проект разрабатывается с использованием гибкой методологии (Agile), включающей:
- Двухнедельные спринты с планированием и ретроспективой
- Ежедневные стендапы для синхронизации команды
- Непрерывная интеграция и развертывание (CI/CD)
- Код-ревью для всех изменений

### 7.2 Управление версиями
- **Семантическое версионирование**: MAJOR.MINOR.PATCH
- **Ветки функциональности**: feature/имя-функции
- **Ветки исправлений**: fix/описание-ошибки
- **Защищенная main-ветка**: Требуется код-ревью перед слиянием

### 7.3 Процесс тестирования
- **Юнит-тестирование**: Тестирование отдельных функций и компонентов
- **Интеграционное тестирование**: Тестирование взаимодействия компонентов
- **End-to-end тестирование**: Тестирование полного пользовательского пути
- **Тестирование производительности**: Оценка скорости работы системы

### 7.4 Процесс развертывания
- **Среда разработки**: Для активной разработки и тестирования
- **Промежуточная среда**: Для предпроизводственного тестирования
- **Производственная среда**: Для конечных пользователей
- **Автоматические и ручные проверки** перед развертыванием в производство
