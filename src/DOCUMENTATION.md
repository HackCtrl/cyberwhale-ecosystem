
# CyberWhale: Документация проекта

## Обзор проекта

CyberWhale - это онлайн-платформа для обучения и практики в области кибербезопасности. Проект создан для объединения студентов, профессионалов и компаний в единую экосистему, способствующую обмену знаниями и развитию навыков в сфере информационной безопасности.

## Структура проекта

### Основные технологии

- **Frontend**: React, TypeScript, Vite
- **Стилизация**: Tailwind CSS
- **UI компоненты**: shadcn/ui (базируется на Radix UI)
- **Состояние и запросы**: TanStack Query (бывший React Query)
- **Анимации**: Framer Motion
- **Маршрутизация**: React Router

## Ключевые компоненты

### Основа приложения

- **App.tsx**: Корневой компонент, содержащий главный маршрутизатор и провайдеры контекста (Auth, Query).
- **main.tsx**: Точка входа в приложение, рендерит App компонент в DOM.

### Компоненты макета

- **Navbar.tsx**: Верхняя навигационная панель, присутствующая на всех страницах.
- **Footer.tsx**: Нижняя часть страницы с информацией о сайте и ссылками.
- **ChatAssistant.tsx**: Компонент ИИ-ассистента, доступный на всех страницах.

### Аутентификация

- **AuthProvider.tsx**: Провайдер контекста аутентификации.
- **LoginForm.tsx**: Форма входа в систему.
- **RegisterForm.tsx**: Форма регистрации.
- **ForgotPassword.tsx**: Форма восстановления пароля.

### Страницы

- **Index.tsx (HomePage)**: Главная страница с обзором возможностей платформы.
- **ctf/index.tsx**: Главная страница CTF платформы со списком заданий.
- **ctf/ChallengePage.tsx**: Страница конкретного CTF задания.
- **knowledge/index.tsx**: Главная страница базы знаний.
- **knowledge/ArticlePage.tsx**: Страница конкретной статьи.
- **ai-assistant/index.tsx**: Страница ИИ-ассистента.
- **community/index.tsx**: Страница сообщества.
- **products/index.tsx**: Страница продуктов.
- **profile/index.tsx**: Страница профиля пользователя.
- **settings/index.tsx**: Страница настроек пользователя.

## База данных и API

Проект использует Supabase в качестве бэкенда для:
- Аутентификации пользователей
- Хранения данных о пользователях, заданиях, статьях и т.д.
- Запросов к API

## Как добавить новое CTF задание

Для добавления нового CTF задания следуйте этим шагам:

1. **Создайте файл с описанием задания**:

```tsx
// В файле src/data/challenges.ts (создайте если его нет)

export interface Challenge {
  id: number;
  title: string;
  description: string;
  category: "web" | "crypto" | "reverse" | "pwn" | "forensics" | "misc";
  difficulty: "easy" | "medium" | "hard";
  points: number;
  solvedBy: number;
  flag: string;
  hints?: string[];
  resources?: { name: string; url: string }[];
}

export const challenges: Challenge[] = [
  // Существующие задания...
  {
    id: 101, // Уникальный ID
    title: "Название задания",
    description: "Подробное описание задания, включая контекст и цель",
    category: "web", // Категория задания
    difficulty: "medium", // Сложность
    points: 200, // Количество очков
    solvedBy: 0, // Начальное количество решивших
    flag: "flag{это_секретный_флаг}", // Флаг, который нужно найти
    hints: [
      "Подсказка 1: Обратите внимание на...",
      "Подсказка 2: Попробуйте использовать..."
    ],
    resources: [
      { name: "Файл задания", url: "/challenges/my_challenge.zip" },
      { name: "Ссылка на задание", url: "https://challenge.example.com" }
    ]
  }
];
```

2. **Добавьте необходимые файлы задания**:
   - Если задание требует файлов, добавьте их в директорию `/public/challenges/`
   - Для веб-заданий настройте соответствующие Docker-контейнеры

3. **Интеграция с бэкендом**:
   - Для динамических заданий добавьте соответствующие записи в базу данных Supabase

```sql
-- Пример SQL для добавления задания
INSERT INTO challenges (id, title, description, category, difficulty, points, solved_by, flag, hints, resources)
VALUES (
  101,
  'Название задания',
  'Подробное описание задания, включая контекст и цель',
  'web',
  'medium',
  200,
  0,
  'flag{это_секретный_флаг}',
  ARRAY['Подсказка 1: Обратите внимание на...', 'Подсказка 2: Попробуйте использовать...'],
  '[{"name": "Файл задания", "url": "/challenges/my_challenge.zip"}, {"name": "Ссылка на задание", "url": "https://challenge.example.com"}]'::jsonb
);
```

## Как добавить новую статью в базу знаний

1. **Создайте файл с описанием статьи**:

```tsx
// В файле src/data/articles.ts (создайте если его нет)

export interface Article {
  id: number;
  title: string;
  description: string;
  content: string;
  category: string;
  tags: string[];
  author: {
    name: string;
    avatar?: string;
  };
  date: string;
  readTime: number;
  imageUrl?: string;
}

export const articles: Article[] = [
  // Существующие статьи...
  {
    id: 201, // Уникальный ID
    title: "Название статьи",
    description: "Краткое описание статьи",
    content: "Полное содержание статьи в формате Markdown или HTML",
    category: "Web Security",
    tags: ["XSS", "CSRF", "Web"],
    author: {
      name: "Имя Автора",
      avatar: "/avatars/author.jpg" // Опционально
    },
    date: "2023-08-15",
    readTime: 10, // Время чтения в минутах
    imageUrl: "/articles/header-image.jpg" // Опционально
  }
];
```

2. **Интеграция с бэкендом**:
   - Добавьте соответствующие записи в базу данных Supabase

```sql
-- Пример SQL для добавления статьи
INSERT INTO articles (id, title, description, content, category, tags, author, date, read_time, image_url)
VALUES (
  201,
  'Название статьи',
  'Краткое описание статьи',
  'Полное содержание статьи в формате Markdown или HTML',
  'Web Security',
  ARRAY['XSS', 'CSRF', 'Web'],
  '{"name": "Имя Автора", "avatar": "/avatars/author.jpg"}'::jsonb,
  '2023-08-15',
  10,
  '/articles/header-image.jpg'
);
```

## Рекомендации по разработке

1. **Организация кода**:
   - Создавайте компоненты в соответствующих директориях
   - Следуйте принципу единой ответственности
   - Выносите повторяющуюся логику в хуки

2. **Производительность**:
   - Используйте мемоизацию (useMemo, useCallback)
   - Оптимизируйте рендеринг списков (виртуализация для больших списков)
   - Следите за лишними ререндерами компонентов

3. **Типизация**:
   - Всегда определяйте типы для props компонентов
   - Используйте интерфейсы для сложных структур данных
   - Избегайте использования any

4. **Стилизация**:
   - Используйте Tailwind CSS для стилизации
   - Следуйте дизайн-системе проекта
   - Создавайте переиспользуемые компоненты для общих UI элементов

## Деплой

Проект развертывается с использованием CI/CD пайплайна:

1. Код пушится в репозиторий
2. CI выполняет проверки (линтинг, тесты)
3. При успешных проверках, код автоматически деплоится
